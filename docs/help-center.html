Help Center Articles

Write and publish documentation with the agent that wrote the code.

The agent that just implemented a feature is the best-placed author for its docs. With the skill, it can draft a help article in Markdown, drop it into the right category, and publish — with revisions tracked so you can restore any earlier version.

Articles have a draft → published → archived lifecycle, SEO fields (seoTitle, metaDescription, urlSlug), multi-language support, and helpful/not-helpful voting.

This guide covers the agent skill and API. For the full product, see Userorbit help center.

Ask your agent

  • Draft a help article explaining how to set up SSO
  • Update the billing article — we renamed the Plans page to Subscription
  • Restore the API keys article to its previous revision

Endpoints

All endpoints are POST https://api.userorbit.com/api/v1/<endpoint> with a Bearer API key and x-team-id header.

EndpointWhat it does
articles.createCreate (title, text, collectionIds)
articles.list / info / searchBrowse and search the knowledge base
articles.update / publish / unpublish / archiveLifecycle management
articles.revisions / restoreVersion history and rollback
article-collections.*Manage help center categories
article-votes.*Read and manage helpful votes

Example

Examplehelp-center
curl -s -X POST "https://api.userorbit.com/api/v1/articles.create" \
  -H "Authorization: Bearer $USERORBIT_API_KEY" \
  -H "Content-Type: application/json" \
  -H "x-team-id: $USERORBIT_TEAM_ID" \
  -d '{"title": "Setting up SSO", "text": "## Overview...", "collectionIds": ["<id>"]}'

Full request and response schemas: references/api.md.
