Getting started

Install once, prompt forever

Add the skill to your agent, export an API key from Settings → API in your Userorbit workspace, and every product chore becomes a one-line prompt.

Read-only keys can call list/info/count/search endpoints; read/write keys can call everything. If your tool speaks MCP, there is also a hosted server at api.userorbit.com/mcp.

Claude Code claude install-skill https://github.com/userorbit/skill
Codex git clone https://github.com/userorbit/skill.git ~/.codex/skills/userorbit
Any agent git clone https://github.com/userorbit/skill.git && point context at userorbit/SKILL.md
Authenticate export USERORBIT_API_KEY="uo_..."
export USERORBIT_TEAM_ID="..."

Why it works

The agent that ships the code closes the loop

Your coding agent already knows what changed — it wrote the diff. The skill gives it a compact map of every Userorbit endpoint, required fields, enums, and workflows, so it can chain calls correctly: find the collection, create the announcement, publish it.

  • Changelog entries written from the actual diff
  • Feedback closed and voters notified when the fix ships
  • Roadmap moves itself — columns are live views over feedback status
  • Help articles drafted by the agent that built the feature
  • Support triaged with your docs and codebase in context
