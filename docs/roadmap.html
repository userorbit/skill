Public Roadmap

Keep your public roadmap in sync with what you actually ship.

Userorbit roadmaps are live views over your feedback: each column is defined by filters (typically feedback status), so the roadmap updates itself as work progresses. Your agent can create roadmap views, define columns, and move items by updating the underlying feedback.

That means "update the roadmap" after a deploy is one prompt: the agent flips the relevant feedback to completed and the public roadmap reflects it immediately.

This guide covers the agent skill and API. For the full product, see Userorbit product roadmap.

Ask your agent

  • Add 'AI-powered search' to the roadmap as planned
  • Move the API v2 item to in progress
  • Create a roadmap view for the mobile app with Planned / Building / Shipped columns

Endpoints

All endpoints are POST https://api.userorbit.com/api/v1/<endpoint> with a Bearer API key and x-team-id header.

EndpointWhat it does
roadmap-views.create / list / infoManage public roadmap views
roadmap-views.items / previewList the feedback resolved into each column
roadmap-views.update / deleteEdit or remove views
roadmap-view-columns.createAdd a column (roadmapId, name, filters)
roadmap-view-columns.update / deleteEdit or remove columns
feedbacks.updateMove an item by changing its status

Example

Exampleroadmap
curl -s -X POST "https://api.userorbit.com/api/v1/roadmap-views.items" \
  -H "Authorization: Bearer $USERORBIT_API_KEY" \
  -H "Content-Type: application/json" \
  -H "x-team-id: $USERORBIT_TEAM_ID" \
  -d '{"id": "<view-id>"}'

Full request and response schemas: references/api.md.
