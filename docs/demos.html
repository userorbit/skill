Interactive Demos

Publish interactive product demos and turn them into documentation.

Interactive demos are pixel-perfect, clickable captures of your product recorded with the Userorbit Chrome extension. Once captured, the skill lets your agent do everything else: polish step copy, publish, duplicate for variants, and read engagement analytics.

The standout: demo-articles.create generates a help center article draft from a demo's captured steps — your demo becomes a how-to guide in one call.

This guide covers the agent skill and API. For the full product, see Userorbit interactive demos.

Ask your agent

  • Publish the checkout demo
  • Generate a help article from the onboarding demo
  • How is the pricing demo performing? Show sessions, completion rate, and drop-off by step

Endpoints

All endpoints are POST https://api.userorbit.com/api/v1/<endpoint> with a Bearer API key and x-team-id header.

EndpointWhat it does
demos.create / list / count / infoManage demos
demos.updateEdit title, description, step copy; publish/unpublish
demos.duplicateClone a demo
demos.publish / unpublish / archiveLifecycle (publish requires privacyReviewAcknowledged)
demo-articles.createGenerate a help article draft from the demo
demos.analyticsSessions, completions, leads, per-step drop-off

Example

Exampledemos
curl -s -X POST "https://api.userorbit.com/api/v1/demos.analytics" \
  -H "Authorization: Bearer $USERORBIT_API_KEY" \
  -H "Content-Type: application/json" \
  -H "x-team-id: $USERORBIT_TEAM_ID" \
  -d '{"id": "<demo-id>"}'

Full request and response schemas: references/api.md.
